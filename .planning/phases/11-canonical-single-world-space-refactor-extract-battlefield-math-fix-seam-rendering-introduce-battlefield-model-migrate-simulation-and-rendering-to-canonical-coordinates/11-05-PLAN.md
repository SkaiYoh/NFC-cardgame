<!-- IMPORTANT: How camera-space rendering works -->
// BeginMode2D(camera) + BeginScissorMode(screenArea) means:
// - The camera sees a region of world space centered at camera.target
// - Camera.rotation rotates the view (90deg for P1, -90deg for P2)
// - Scissor clips to the screen rectangle
// - entity_draw(e) uses e->position which is in WORLD space
// - The camera transforms world->screen automatically via BeginMode2D

// After migration: entities store canonical positions (0..1080 x 0..1920).
// P1's camera targets the BOTTOM territory center.
// P2's camera targets the TOP territory center.
// The camera's View matrix handles the coordinate->screen conversion.
// So we can draw entities at their canonical position directly!
// No bf_to_local needed for MOST entities -- BeginMode2D handles it.

// The simplest approach: for each viewport, draw ALL entities and let
// the scissor rect clip anything outside the viewport screen area.
// The GPU handles the clipping efficiently.
// entity_draw() calls sprite_draw(e->sprite, &e->anim, e->position, e->spriteScale)
// e->position is canonical. BeginMode2D(p->camera) transforms it to screen.

    **Step A: Rewrite game_draw_entities_for_viewport in game.c**

    Replace the entire `game_draw_entities_for_viewport` function with a simplified version that iterates the Battlefield's single entity list:

    ```c
    // Draw all entities visible in this viewport.
    // In canonical space, both cameras view the same world. Each camera+scissor
    // clips to one territory, so we draw ALL entities and let the scissor handle
    // visibility. Entities near the seam are drawn in BOTH viewports naturally.
    static void game_draw_entities_for_viewport(GameState *g) {
        Battlefield *bf = &g->battlefield;

        for (int i = 0; i < bf->entityCount; i++) {
            const Entity *e = bf->entities[i];
            if (!e || !e->alive || !e->sprite) continue;

            #ifndef NDEBUG
            CanonicalPos epos = { e->position };
            BF_ASSERT_IN_BOUNDS(epos, bf->boardWidth, bf->boardHeight);
            #endif

            entity_draw(e);
        }
    }
    ```

    That's it. Since entities are in canonical coordinates and each viewport's Camera2D is set up to view the correct territory, BeginMode2D handles the world-to-screen transform. The scissor rect clips entities that are out of view. Entities near the seam will be partially visible in both viewports because the scissor clips per-pixel.

    Note: this approach means entity_draw is called for ALL entities in BOTH viewports (2 * entityCount draw calls per frame). With MAX_ENTITIES*2 = 128, that's 256 draw calls, which is fine per the existing TODO comment.

    Remove the `const Player *viewportPlayer` parameter since it's no longer needed.

        const Color laneColors[3] = {BLUE, GREEN, RED};

        // Draw waypoints for both sides in their respective player's screen-space camera
        for (int s = 0; s < 2; s++) {
            Camera2D cam = gs->players[s].camera;
            BattleSide side = gs->players[s].side;

            for (int lane = 0; lane < 3; lane++) {
                Color c = laneColors[lane];
                for (int i = 0; i < LANE_WAYPOINT_COUNT; i++) {
                    Vector2 wp = bf->laneWaypoints[side][lane][i];
                    Vector2 sp = GetWorldToScreen2D(wp, cam);
                    DrawCircleV(sp, 4.0f, c);

                        Vector2 spNext = GetWorldToScreen2D(wpNext, cam);
                        DrawLineV(sp, spNext, c);
                    }
                }
            }
        }
    }
